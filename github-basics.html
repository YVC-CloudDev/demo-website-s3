GitHub Basics for students

Goal: push code confidently and collaborate without breaking main branch.

Learning objectives

  • Understand repo, commit, branch, and pull request
  • Use a simple feature branch workflow
  • Write a basic README for your project

Safe workflow for beginners

  1. Create a new branch for each task
  2. Commit small changes with clear messages
  3. Open a pull request into main
  4. Merge only when checks pass

Starter command sequence

Run this from your project folder:

git init git add . git commit -m "Initial commit" git branch -M main git remote add origin YOUR_REPO_URL git push -u origin main

Common mistakes and fixes

Mistake: giant commits

Fix: commit each small lesson step separately so your history is readable.

Mistake: editing directly on main

Fix: always create a branch first, even for tiny updates.

Mistake: no README

Fix: include setup, architecture, and screenshots for recruiters and teammates.

Mistake: unclear commit messages

Fix: use patterns like "Add S3 bucket policy section" or "Fix CloudFront path".
